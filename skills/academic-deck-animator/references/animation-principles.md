# 學術簡報動畫使用原則

動畫在學術場合是雙面刃:用得準,聽眾的注意力被你牽著走;用過頭,口試委員第一個
念頭是「這是 AI 做的吧」,第二個念頭是「內容是不是也是 AI 湊的」。本文件是紅線與
節拍設計指南,產生任何 slides_content.json 前先讀完。

## 三條紅線(違反即拒絕,並向使用者解釋)

1. **結果不做秀**:迴歸表、係數、p 值、統計圖,禁止任何裝飾性動畫(旋轉、彈跳、
   縮放進場)。唯一允許:逐欄/逐列/逐條「揭示」,而且每步必須對應講者的一句話。
2. **粒子特效限定三處**:封面(cover)、章節分隔頁(section_divider)、結尾頁
   (closing)。內容頁出現粒子=聽眾眼睛離開你的論證。引擎在其他版型會忽略 fx
   並警告,這是刻意設計,不要繞過。
3. **一頁最多 5 個動畫步**:超過就是「連環動畫」,是 AI 簡報最典型的破綻
   (ppt-master 專案的原話:unsolicited cascade reads as the AI deck tell)。
   內容多就分頁,不是加步數。

## 節拍設計方法:先寫講稿,再配動畫

正確順序永遠是:內容 → 講稿停頓點 → 動畫。反過來(先想動畫再湊內容)必然過度設計。

具體做法:
1. 對每一頁,請使用者(或代替使用者)用一句話說出「這頁要讓聽眾記住什麼」。
2. 把這頁的講解拆成 2–4 個「說話節拍」(講者自然停頓、看聽眾一眼的地方)。
3. 每個節拍對應一個 animation_step / entry_order。節拍之外,零動畫。
4. `trigger` 預設 on-click:講者自己控制節奏,永遠比自動播放安全。
   `after-previous` 只用在同一節拍內的視覺分層(例:框先出現、標籤 200ms 後跟上)。

## 各版型的建議模式

| 版型 | 建議動畫 | 反面教材 |
|---|---|---|
| cover | fx 一種、低強度;標題不動畫或單次 fade | 標題飛入+粒子+轉場三重奏 |
| bullet_points | 逐條 fade-in,300–500ms | 每條不同方向飛入 |
| architecture_chart | 依論證順序搭建:構念→構念→假說箭頭 | 全圖同時飛入(等於沒動畫)或逐字出現 |
| two_column | 左欄整體→右欄整體,兩步 | 左右交錯逐條 |
| figure_focus | 不動畫,或整圖單次 fade | 對圖做 zoom/pan 秀 |
| section_divider | fx 低強度或無;編號+標題一次出現 | — |
| closing | 同 cover | — |

## 效果選擇的語意

- `appear` / `fade-in`:中性,萬用。猶豫時選 fade-in。
- `wipe`:有「方向感」的揭示,適合時間軸、流程、連接線。
- `fly-in-*`:動作最大,一頁最多一次,用在真正的重點落點(例:研究缺口那一格)。
- 沒有 bounce、spin、zoom──不是引擎做不到,是學術場合不該有,所以不提供。

## 口試情境的特別提醒

- 委員可能要求「回到上一頁」:native-pptx 動畫在回放時會重新觸發,若委員反覆前後翻,
  動畫會變成干擾。**口試簡報建議準備 --split 拆頁版當備用**,現場可即時切換。
- 交給主辦方/口委的 PDF 講義一律用拆頁版轉出,確保每個中間狀態都看得到、排版不重疊。
- 現場設備未知時,html-canvas 版本自帶一切依賴,隨身碟+任何瀏覽器就能播,是最穩的 Plan B。
