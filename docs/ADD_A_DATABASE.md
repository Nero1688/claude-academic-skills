# 如何新增一個資料庫 Profile / Add a Database Profile

`tej-data-scout` 的方法（拆構念 → 路由模組 → 檢核可得性 → 建議方法）**適用於任何資料庫**。歡迎你為自己常用的資料庫（WRDS/Compustat/CRSP、CSMAR、Wind、World Bank、公開資訊觀測站、政府開放資料…）新增一份 profile 並分享回來。

## 一條不可跨越的紅線
**Profile 只描述「導覽方法」與「指向官方公開文件」，絕不重製資料庫的專屬完整目錄、欄位代碼全表或需訂閱才能看到的內容。**

- ✅ 可以：「在 WRDS 進 Compustat → Fundamentals Annual，用 `at`(總資產) 這類公開文件可查到的欄位名。」並附 WRDS 官方文件連結。
- ✅ 可以：整理自廠商**公開官網**的模組分類、公開範例檔的欄位名。
- ❌ 不可以：把你從**訂閱終端機**匯出的完整導航樹、內部欄位字典整包貼上。
- ❌ 不可以：標註「內容取自某機構的訂閱介面」之類的來源——這會連累你的機構。

## Profile 檔案結構（建議）
在 `skills/tej-data-scout/references/` 下新增 `〔資料庫〕-catalog.md`，包含：

1. **檔頭聲明**：資料來源僅限公開官網／公開文件；確切欄位以使用者自己的訂閱終端機為準；附官方文件連結。
2. **Part A：五大／N 大資料集 → 模組 路由地圖**（語義分類即可）。
3. **Part B：模組 → 代表性資料表 索引**（表名 + 一句用途；不貼完整欄位表）。
4. **Part C：研究構念 → 模組 速查**（最實用，例：Tobin's Q → 財務 + 市值模組）。
5. **Part D：時間範圍／頻率／樣本範圍 注意事項 + 欄位驗證方法**（怎麼在官方文件或範例檔確認）。

> 不需要、也**不應該**放「實際訂閱終端機的完整導航樹」。給**方向**，讓使用者在自己的帳號裡確認。

## 提交步驟
1. 依上述結構寫好 `〔資料庫〕-catalog.md`。
2. 若你參考了任何他人作品，附 `ATTRIBUTION.md` 並保留原授權；更新 `NOTICE.md`。
3. 在 repo 根目錄執行 `bash scripts/sanitize_check.sh`，確認沒有個資、機構帳號、金鑰。
4. 送 PR，說明資料來源皆為公開文件。

謝謝你讓這個框架對更多研究者有用 🙏
